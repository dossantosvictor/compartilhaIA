# Agentic Squad Framework

### Transformando Software Delivery com AI-Native Engineering

---

## Vision

Da ideia ao deploy com especificação como fonte única de verdade.

Traditional squads execute tasks.

Agentic squads orchestrate intelligence.

---

# O Problema Atual

O modelo tradicional de desenvolvimento não escala bem com a complexidade crescente.

Principais desafios:

- Contexto se perde entre Produto e Engenharia
- Refinamentos excessivos e muito esforço manual
- Código começa antes da especificação estar madura
- Modernização gera retrabalho constante
- Débito técnico aumenta em projetos legados
- Conhecimento concentrado em poucas pessoas

---

# Mudança de Paradigma

## Modelo Tradicional

Developer + AI Tool

IA apenas auxilia desenvolvimento.

---

## Modelo Agentic

Humans + Autonomous Specialist Agents

IA deixa de ser assistente e passa a atuar como força operacional.

---

# Framework Operacional

## 4 Fases

DISCOVER → MODEL → SPECIFY → DELIVER

---

## Phase 1 — Discover

Frameworks utilizados:

- Lean Inception
- Product Backlog Building (PBB)

Objetivos:

- Definir visão do produto
- Entender problema
- Criar backlog inicial
- Construir User Stories

Outputs:

/docs/product/vision.md  
/docs/product/problem.md  
/docs/product/backlog.md  
/docs/product/user-stories.md  

---

## Phase 2 — Model

Frameworks utilizados:

- Domain Driven Design (DDD)
- Event Storming

Objetivos:

- Mapear domínio
- Criar bounded contexts
- Definir arquitetura
- Construir linguagem ubíqua

Outputs:

/docs/architecture/context-map.md  
/docs/architecture/glossary.md  
/docs/architecture/system-diagram.md  

---

## Phase 3 — Specify

Framework utilizado:

- Spec Driven Development (SDD)

Objetivos:

- Transformar user stories em especificações técnicas
- Criar critérios de aceite
- Gerar tasks técnicas

Outputs:

/specs/0001-feature/spec.md  
/specs/0001-feature/tasks.md  
/specs/0001-feature/contracts.md  
/specs/0001-feature/acceptance.md  

---

## Phase 4 — Deliver

Objetivos:

- Implementação assistida por IA
- Validação automatizada
- Deploy incremental seguro

Outputs:

- Código
- Testes
- CI/CD
- Métricas
- Deploy incremental

---

# Human Approval Gates

O fluxo mantém controle humano em decisões críticas.

---

## Gate 1 — Product Approval

Responsáveis:

- Product Manager
- Tech Lead

Aprovação de:

- Product Vision
- Priorização do Backlog
- User Stories

---

## Gate 2 — Architecture Approval

Responsável:

- Tech Lead

Aprovação de:

- Domain Model
- Bounded Contexts
- Architecture Mapping

---

## Gate 3 — Engineering Approval

Responsável:

- Engineer / Tech Lead

Aprovação de:

- Technical Specification
- Acceptance Criteria
- Generated Tasks

Nenhuma linha de código começa antes desse gate.
